This is Softheon's baseline Angular6 and .NET Core 2.1 application, to be used for starting a new web application project.

This project has been pre-configured with a number of Softheon-chosen settings for both the .NET CORE API and the Angular6 portions of the project.

To see a full set of base settings, please go to Release Notes.

Below you will also find helpful links to help understand the project. Please remember to follow the Angular CLI standards when creating new Angular files.

Setting up your new project

There are some steps you should follow when starting a new project with this template:

  • Use npm-check-updates
    • This is a command line tool that lets you see if you have the latest package versions of he npm packages included within this project.
    • Be careful about updating all packages to the latest version, as sometimes other packages may not be compatible with, for example, the latest version of typescript.
    • All packages have been set to not automatically use automatic matching of package versions, as packages we use may not follow the standards to upgrade the major version if any breaking changes are found.
    • This allows the developer to better ensure that the package versions included in this base package have been tested and confirmed to work without causing issues in building a base starting project.
  • Authorized Login
    • In the appsettings.json you can determine if the website should be gated based on the user being able to successfully log in to identity server
    • If you want authorization, set the realm and other necessary information in both UI/Auth and Identity sections of the appsettings.json
    • If you are certain your application will either never, or always, require validation, you can remove the references to AuthIsRequired from both the C# and the Angular code, including in startup.service.ts
    • If you are not using a realm, you may remove references to the realm from the configurations, but make sure you are certain before committing to this.
  • Softheon Workshop (Below references are in alpha and are subject to change)
    • The project is set up to use @softheon/ng-workshop as a Softheon component library. If you are unable to download this library, please ensure you have an account on the Softheon npm
    • The alpha is published to our private Softheon NPM repo and can be installed via:
      npm i @softheon/ng-workshop
    • Components include radio buttons, check boxes, buttons, links, toggle buttons, tooltips, and more.

Generating a template

  1. Download Visual Studio SDK
    • Use Visual Studio Installer to include ability to create Visual Studio Extensibility projects
  2. Generate a .zip folder
    • In Visual Studio, highlight the project and in the top menu go to Project > Export Template...
    • Choose Project Template
    • Set the project name to "Softheon Templates"
    • Set the description to "UI Project Template using Angular and .NET CORE API"
    • Un-check the checkbox to import the template directly into your Visual Studio
    • This will generate a zip file in your system called Softheon Templates.zip
  3. Update .vstemplate
    • Extract the Softheon Templates.zip folder
    • Open the MyTemplate.vstemplate file
    • Ensure that the following code is the last entry in >Template Data< : >CreateInPlace<true>/CreateInPlace<
    • The above step is based on this link as including this prevents a bug where items are missing from the final .vsix
  4. Update Launch Settings
    • Extract the Softheon Templates.zip folder
    • Open the Properties/launchSettings.json
    • Remove any lines showing "applicationUrl"
    • This ensures that the port number for the base project will not be carried over into the newly-generated UI project.
  5. Import into AngularApi.Vsix
    • Copy over the Softheon Templates.zip file in the AngularApi.Vsix project
    • Also copy over the file of the same name in the Project Templates folder
    • Update the author and version number in source.extension.xsixmanifest
  6. Test project
    • Set the .Vsix project as the startup project and run it
    • A new instance of Visual Studio will open.
    • From this instance of Visual Studio, Select to create a New Project
    • If the above steps were followed correctly the project type should appear under Visual C# in the folder Softheon Templates
    • Note that if the project appears as a subfolder within Softheon Templates instead, then you need to update the .zip file to ensure there are no subfolders holding the .csproj file
    • Create the project, build and wait for dependencies to install, and run. The project should build correctly.
  7. Distribute .vsix file
    • Build the .Vsix project once you are sure it is ready.
    • In the bin folder generated, you will find a .vsix file
    • Distribute this .vsix file to whoever should have the template. Don't forget to install it yourself!
    • The template should now display for you when making a new project
    • To un-install, go to Tools > Extensions and Updates... > Templates
